## Project

The Accountability Project is an effort to cut across data silos and
give journalists, policy professionals, activists, and the public at
large a simple way to search across huge volumes of public data about
people and organizations.

Our goal is to standardizing public data on a few key fields by thinking
of each dataset row as a transaction. For each transaction there should
be (at least) 3 variables:

1.  All **parties** to a transaction.
2.  The **date** of the transaction.
3.  The **amount** of money involved.

## Objectives

This document describes the process used to complete the following
objectives:

1.  How many records are in the database?
2.  Check for entirely duplicated records.
3.  Check ranges of continuous variables.
4.  Is there anything blank or missing?
5.  Check for consistency issues.
6.  Create a five-digit ZIP Code called `zip`.
7.  Create a `year` field from the transaction date.
8.  Make sure there is data on both parties to a transaction.

## Data

Data is obtained from the [Nebraska Accountability and Disclosure
Commission (NADC)](http://www.nadc.nebraska.gov/).

> This site includes the campaign statements filed with the Commission
> by candidates for state and local offices, political action committees
> or PACs, political parties, and ballot question committees. Campaign
> statements disclose the amount and sources of contributions to the
> committee and the amount purposes of expenditures by the committee.

The data itself can be downloaded from the [Campaign Statements
Search](http://www.nadc.nebraska.gov/ccdb/search.cgi) page, where there
is a [direct download
link](http://www.nebraska.gov/nadc_data/nadc_data.zip).

### Read

The records of interest are spread across a number of seperate files,
the descriptions of which are contained in the `nadc_tables.rtf` file.
Furthermore, the variables describing filing committees are contained in
additional files. We find read all these files and use a combination of
`dplyr::left_join()` and `dplyr::bind_rows()` to create a single table
with all the data we want.

From reading the tables document, it seems like these files would
contain the relevant data:

  - `FORMA1` – Statement of Organization of a Political Committee:
  - `FORMB1` – Campaign Statement for Candidate and Ballot Question
    Committee:
  - `FORMB1AB` – All Data from form B-1 schedule A and B, Contributions:
      - `Committee ID` – Use to link with `FORMb1`, along with Date
        Received
  - `FORMB2` – From B-2 Campaign Statement Political Party Committees:
  - `FORMB2A` – Form B-2 Contributions:
  - `FORMB4` – Campaign Statement for Independent Committees:
  - `FORMB4A` – Form B-4 Schedule A:
      - `Committee ID` - Use to link with `FORMB4`, along with Date
        Received
  - `FORMB5` – Report of Late contributions:
  - `FORMB7` – Report of Political Contributions of Corporation, Union
    or Other:
  - `FORMB72` – Form B-7 Direct contributions:
  - `FORMB73` – Form B-7 Indirect contributions:
  - `FORMB9` – Out of State Contribution Report:
  - `FORMB9A` – Form B-9 Schedule A, contributions:
      - `Form B9 ID` – Use to link with `FORMB9`

First we will read *every* file into a list of data frames with
`purrr::mpa()` and `vroom::vroom()`.

``` r
nadc <- map(
```
